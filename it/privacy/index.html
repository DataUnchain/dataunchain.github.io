🔒 Privacy

Privacy Policy

Ultimo aggiornamento: Marzo 2026

🛡️ La Versione Breve

DataUnchain non raccoglie, trasmette o archivia nessuno dei tuoi dati. Tutto gira sul tuo hardware. Nessuna telemetria, nessun analytics, nessuna chiamata cloud — mai. I tuoi documenti non lasciano mai la tua macchina.

1. Informazioni su DataUnchain

DataUnchain ("Software") è una soluzione aziendale di estrazione dati AI da documenti, self-hosted. È sviluppato e mantenuto da Antonio Trento.

Il Software è progettato per funzionare interamente offline sull'hardware dell'utente tramite Docker. Nessuna connessione internet è necessaria dopo la configurazione iniziale.

2. Raccolta Dati dal Software

Nessuna. DataUnchain:

  • Non raccoglie dati personali
  • Non invia telemetria o analytics di utilizzo
  • Non effettua chiamate di verifica licenza online
  • Non trasmette il contenuto dei documenti a server esterni
  • Non richiede una connessione internet per funzionare

Tutta l'inferenza AI (il nostro Vision Language Model proprietario via Ollama) avviene localmente nei container Docker sulla tua macchina. I dati estratti sono salvati in un database PostgreSQL locale e in export locali — tutto all'interno della tua infrastruttura.

3. Architettura di Trattamento Dati

L'architettura di DataUnchain è privacy by design:

  • Air-Gapped: L'intero stack (Ollama + Processor + PostgreSQL) gira in Docker senza accesso di rete in uscita
  • Zero Dipendenze Cloud: Nessun AWS, Google Cloud, OpenAI o API esterna utilizzata
  • Nessuna Condivisione con Terzi: I tuoi documenti e dati estratti restano sul tuo hardware
  • Storage Controllato dall'Utente: Tutti gli output (JSON, Excel, PostgreSQL) sono in volumi che controlli tu

4. Questo Sito Web (dataunchain.com)

Questo sito è un sito statico ospitato su GitHub Pages. Può utilizzare analytics di base (es. analytics di traffico integrati di GitHub). Nessun cookie viene impostato dal sito stesso oltre a quelli previsti da GitHub Pages.

5. Conformità GDPR

Poiché DataUnchain elabora dati esclusivamente sulla tua infrastruttura, senza uscita di dati dal tuo dispositivo:

  • Nessun Data Processing Agreement (DPA) è necessario con l'autore
  • Nessun trasferimento dati avviene — conforme ai requisiti di residenza dati UE
  • L'utente è sia titolare che responsabile del trattamento
  • Nessun sub-responsabile è coinvolto

6. Contatti

Per domande su questa policy, contatta il maintainer del progetto tramite il GitHub issue tracker.
