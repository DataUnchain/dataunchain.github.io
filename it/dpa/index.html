Perché DataUnchain non necessita di un DPA

🛡️ Nessun DPA Necessario

DataUnchain è 100% self-hosted. Nessun dato viene trasmesso all'Autore o a terze parti. Non esiste alcun rapporto di trattamento dati — sei sia il titolare che il responsabile del trattamento. Un Data Processing Agreement non è pertanto applicabile.

Qual è la differenza con i servizi cloud?

Quando usi un servizio cloud di elaborazione documenti (AWS Textract, Google Doc AI, ecc.), i tuoi dati vengono inviati ai loro server. Sotto il GDPR, questo crea un rapporto di trattamento dati che richiede un DPA.

Con DataUnchain:

  • ✅ Tutta l'inferenza AI gira sul tuo dispositivo
  • ✅ I tuoi documenti restano sul tuo dispositivo
  • ✅ I dati estratti sono salvati sul tuo dispositivo
  • ✅ Nessuna connessione di rete necessaria
  • ✅ Nessun sub-responsabile coinvolto

Per i settori regolamentati

DataUnchain è particolarmente adatto per settori dove i dati non possono uscire dalla sede:

  • Sanità: Cartelle cliniche, referti — deploy air-gapped
  • Legale: Segreto professionale preservato — zero esposizione cloud
  • Finanza: Segreti commerciali, documenti di compliance — sovranità totale sui dati
  • Pubblica Amministrazione: Documenti classificati o sensibili — deploy on-premise

Per domande, contatta il maintainer del progetto tramite il GitHub issue tracker.
