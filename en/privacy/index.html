🔒 Privacy

Privacy Policy

Last updated: March 2026

🛡️ The Short Version

DataUnchain does not collect, transmit, or store any of your data. Everything runs on your hardware. No telemetry, no analytics, no cloud calls — ever. Your documents never leave your machine.

1. About DataUnchain

DataUnchain ("Software") is an enterprise, self-hosted AI document data extraction engine. It is authored and maintained by Antonio Trento.

The Software is designed to run entirely offline on the user's own hardware using Docker. No internet connection is required after initial setup.

2. Data Collection by the Software

None. DataUnchain:

  • Does not collect personal data
  • Does not send telemetry or usage analytics
  • Does not phone home or verify licences online
  • Does not transmit document contents to any external server
  • Does not require an internet connection to operate

All AI inference (Qwen 3.5 VL via Ollama) runs locally inside Docker containers on your machine. Extracted data is saved to a local PostgreSQL database and local file exports — both within your own infrastructure.

3. Data Processing Architecture

DataUnchain's architecture is privacy by design:

  • Air-Gapped: The entire stack (Ollama + Processor + PostgreSQL) runs inside Docker with no outbound network access required
  • No Cloud Dependencies: No AWS, Google Cloud, OpenAI, or any external API is used
  • No Third-Party Data Sharing: Your documents and extracted data stay on your hardware
  • User-Controlled Storage: All output (JSON, Excel, PostgreSQL) is stored in volumes you control

4. This Website (dataunchain.com)

This website is a static site hosted on GitHub Pages. It may use basic analytics (e.g., GitHub's built-in traffic analytics). No cookies are set by the site itself beyond what GitHub Pages provides by default.

External resources loaded by this site (e.g., Google Fonts, Font Awesome, Tailwind CSS CDN) are subject to their respective providers' privacy policies.

5. GDPR Compliance

Because DataUnchain processes data exclusively on your infrastructure, with no data leaving your device:

  • No Data Processing Agreement (DPA) is required with the author
  • No data transfers occur — compliant with EU data residency requirements
  • The user is both the data controller and data processor
  • No sub-processors are involved

For questions about this policy, contact the project maintainer via the GitHub issue tracker.
